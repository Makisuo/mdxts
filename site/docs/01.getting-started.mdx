# Getting Started

<Summary>

Build interactive, type-safe content and documentation using MDX,
TypeScript, and React.

MDXTS generates a graph of prose content, source code, type information, and examples that can be rendered through components and utilities using [React 18 Server Components](https://react.dev/blog/2023/03/22/react-labs-what-we-have-been-working-on-march-2023#react-server-components).

</Summary>

<Note>
  Server Component support is currently limited in frameworks. This library
  offers first-class support for Next.js through a [plugin](packages/next).
  Please open an issue or PR to add support for other frameworks.
</Note>

## Install

Run the following command in your project directory to install the required dependencies:

<PackageInstall
  packages={['mdxts', 'next', 'react', 'react-dom', 'shiki', 'prettier']}
/>

## Guided Setup

### Configure Plugin

Once the `mdxts` package is installed, you can configure the Next.js plugin to use a custom theme and git source:

```ts filename="next.config.ts"
import { createMdxtsPlugin } from 'mdxts/next'

const withMdxts = createMdxtsPlugin({
  theme: 'theme.json',
  gitSource: 'souporserious/mdxts',
})

export default withMdxts({
  // Next.js config options...
})
```

### Type Checking

All code blocks will automatically be type-checked using the TypeScript compiler. You can also use the `types` plugin option to specify additional package types to include for the client:

```ts filename="next.config.ts"
import { createMdxtsPlugin } from 'mdxts/next'

const withMdxts = createMdxtsPlugin({
  theme: 'theme.json',
  gitSource: 'souporserious/mdxts',
  types: ['mdxts/components', 'react', 'react-dom'],
})
```

### Generating Routes

Use the `createDataSource` function exported from the `mdxts` package to generate metadata and paths from a set of MDX or TypeScript files:

```ts filename="data.ts"
import { createDataSource } from 'mdxts'

export const allDocs = createDataSource('docs/**/*.mdx', {
  baseDirectory: 'docs',
})

allDocs.paths() // => ['docs/getting-started.mdx', 'docs/authoring.mdx']
```

### Rendering Content

Use the `get` helper from the modules loader to render MDX content:

```tsx filename="page.tsx"
import { notFound } from 'next/navigation'
import { allDocs } from './data'

type Props = { params: { slug: string } }

export default async function Page({ params }: Props) {
  const doc = await allDocs.get(params.slug)

  if (doc === null) {
    return notFound()
  }

  const { Content } = doc

  return <Content />
}
```
