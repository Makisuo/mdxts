# Getting Started

<Summary>

Build interactive, type-safe content and documentation using MDX and
TypeScript.

MDXTS generates a graph of prose content, source code, and examples that
can be rendered through components and utilities in MDX and React.

</Summary>

## Install

The `mdxts` library can be installed using your favorite package manager:

```bash
npm install mdxts
pnpm install mdxts
yarn add mdxts
```

This library includes exports for various use cases that work best when used together, but can also be used independently.

## Guided Setup

### Configure Plugin

Once the `mdxts` package is installed, you can configure the Next.js plugin to use a custom theme and git source:

```ts filename="next.config.ts"
import { createMdxtsPlugin } from 'mdxts/next'

const withMdxts = createMdxtsPlugin({
  theme: 'theme.json',
  gitSource: 'souporserious/mdxts',
})

export default withMdxts({
  // Next.js config options...
})
```

### Enable Type Checking

To enable type checking, you need to add a `tsconfig.json` file to your project. The `mdxts` package will

```ts filename="next.config.ts"
import { createMdxtsPlugin } from 'mdxts/next'

const withMdxts = createMdxtsPlugin({
  theme: 'theme.json',
  gitSource: 'souporserious/mdxts',
  types: ['mdxts/components', 'react', 'react-dom'],
})
```

### Generating Routes

Use the `loadModules` function exported from the `mdxts` package to generate metadata and paths from a set of modules:

```ts filename="data.ts"
import { loadModules } from 'mdxts'

export const allDocs = loadModules(
  require.context('./docs', true, /\.mdx$/),
  'docs'
)
```

### Rendering Content

Use the [[get]] helper from the modules loader to render MDX content:

```tsx filename="page.tsx"
import { notFound } from 'next/navigation'
import { allDocs } from './data'

export default async function Page({ params }) {
  const doc = await allDocs.get(params.slug)

  if (doc === null) {
    return notFound()
  }

  const { Component } = doc.active

  return <Component />
}
```
